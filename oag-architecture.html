🏗️ 四层架构设计

Layer 4: 推理预测层 Reasoning
  • Agent + 工具调用 + 多跳推理 + 权限感知回答 + 预测告警
Layer 3: 带权限融合图谱层 Permissioned KG
  • Neo4j + RBAC 权限模型 + 实体消歧融合 + 溯源追踪
Layer 2: 摘要结构化层 Summarization
  • 分块 → 实体/关系/事件抽取 → 文档摘要 → bge-m3 Embedding
Layer 1: 原始数据层 Raw Storage
  • MinIO (文件) + PostgreSQL (元数据)，不可变存储，SHA-256 校验

🔧 技术选型建议

🟢 推荐方案

  • L1 存储: MinIO + PostgreSQL
  • L2 提取: Cognee (6行代码) 或 自定义流水线
  • L3 图谱: Neo4j 社区版 (Cypher + 权限)
  • L3 向量: Milvus 或 Neo4j Vector
  • L4 Agent: LangGraph 或 自定义
  • 权限: RBAC (角色级) → ABAC
  • Embedding: bge-m3 (中文首选)
  • 音视频: WhisperX (自部署不出境)
  • 文档: Unstructured.io (格式最全)

🔴 避坑指南

  • 纯向量 RAG: 无法做多跳推理
  • 从零写图谱: 6000行 vs Cognee 6行
  • 个人 Wiki: 撑不住企业权限/多模态
  • ABAC 起步: 一开始就是噩梦
  • 全量图谱 Day 1: 从文档+邮件开始
  • 小模型抽取: 垃圾进 → 垃圾图谱

🤖 Agent 集成方案

oag_search: 权限感知查询工具

1. 获取 user_roles
2. neo4j.query( inject_acl(cypher, roles) )
3. milvus.search( query, filter=roles )
4. 融合排序 (Graph + Vector)
5. 附加 L1 原始溯源链接

多跳推理

"张总上周提到的项目进展如何？" 自动关联人物-时间-事件

权限感知

越权信息静默过滤，只返回用户有权看的内容

溯源底线

每条结论必须追溯到 L1 原始数据，拒绝赛博背锅
