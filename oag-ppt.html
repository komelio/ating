最新知识库相关技术全景

LLM Wiki

颠覆传统RAG查询时检索模式，采用摄入时编译模式，三层架构设计，知识完全可读可审计、可版本管理，无黑盒问题。

发布者：Andrej Karpathy | 2026年4月

GBrain

工程化知识操作系统，支持全多模态数据源，双存储设计（当前结论+证据时间线），来源可追溯，全链路自动化。

发布者：Garry Tan | 开源项目

配套前沿技术

Agentic RAG：智能体增强检索
Graph RAG：知识图谱增强检索
深度混合检索：向量+关键词+元数据

OAG知识体系四层架构总览

🔹 原始层

不可变原始数据源，唯一事实源，保障可追溯性

🔹 原始层摘要层

多模态知识提取、摘要、结构化

🔹 带权限融合图谱层

知识互联互通 + 细粒度权限控制

🔹 推理预测层

支持Agent复杂推理 + 知识自进化

第一层：原始层（Raw Layer）

核心职责：存储所有不可变的原始数据源，作为知识的唯一事实源，保障可追溯性

📁 支持的数据源

✅ 结构化：数据库、Excel
✅ 半结构化：邮件、聊天记录
✅ 非结构化：文档、音视频、图片

⚙️ 技术选型

对象存储：MinIO/OSS
关系数据库：PostgreSQL
日志系统：ELK

✅ 核心特性

只读不可修改
版本管理
全操作日志审计

第二层：原始层摘要层（Abstract Layer）

核心职责：完成从原始数据到结构化知识的转化，对原始数据进行预处理、提取、摘要、标记

核心处理流程

1. 多模态解析
2. 知识提取
3. 摘要生成
4. 元数据标记

技术选型

多模态解析：RAGFlow + Whisper + Qwen-VL
支持所有格式，包括扫描件、音视频
知识提取：DeepSeek V4/Qwen 2.5 72B
实体、关系、属性抽取
存储：Git（Markdown摘要） + LanceDB（向量+元数据）

第三层：带权限融合图谱层

核心职责：构建企业级统一知识图谱，实现知识互联互通，同时嵌入细粒度权限控制，保障知识安全

核心能力

✅ 知识图谱构建、实体融合、关系补全
✅ 细粒度权限控制（RBAC+ABAC）
✅ 混合检索：向量+关键词+图谱

技术选型

知识图谱：Nebula Graph
分布式万亿级知识图谱支持
权限引擎：OPA（Open Policy Agent）
检索引擎：Elasticsearch + LanceDB + Nebula

第四层：推理预测层

核心职责：基于知识库提供推理能力，支撑Agent的复杂任务执行，同时实现知识的自主进化

核心能力

✅ 精准问答、多轮会话
✅ 多跳推理、跨文档综合分析
✅ 趋势预测、风险预警
✅ 知识自校验、自更新、自进化

技术选型

大模型服务：vLLM + 豆包4.0/DeepSeek V4
完全私有化部署，支持主流模型
Agent框架：OpenClaw Agent/LangGraph
自校验模块：LLM自动校验+人工审核

知识提取核心流程设计

1. 数据源接入：API定时拉取、webhook实时推送、人工手动上传

2. 预处理：格式转换 → 去重 → 清洗（去除冗余、乱码、无效内容）

3. 知识提取：实体抽取 → 关系抽取 → 属性抽取 → 标签提取

4. 知识融合：实体对齐 → 关系补全 → 冲突检测（标记矛盾知识，触发人工审核）

5. 落库存储：向量库 + 图谱库 + 摘要库 + 原始库同步更新

Agent集成方案

核心定位

知识库作为Agent的核心记忆模块：
👉 短期记忆：会话上下文，存储在Agent会话状态中
👉 长期记忆：OAG知识库，执行任务时自动检索，自动过滤无权限内容

交互流程

1. 任务接收
2. 知识检索
3. 推理执行
4. 知识回写
5. 结果返回

典型应用场景

智能客服 | 研发助手 | 会议助理 | 决策助手

落地建议

1. 小步快跑，快速验证：先从核心业务场景切入（如研发知识库、客服知识库），验证效果后逐步扩展

2. 优先选择私有化开源工具：保障企业数据安全，避免供应商锁定

3. 权限设计循序渐进：先从粗粒度部门级权限开始，逐步细化到节点级、属性级细粒度权限

4. 建立质量保障机制：关键知识人工审核，确保知识准确性

5. 持续迭代优化：收集用户反馈，持续优化提取、检索、回答准确率
